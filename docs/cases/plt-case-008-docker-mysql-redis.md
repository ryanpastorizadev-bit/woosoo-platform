## Contrarian Review

**Tier escalated: 2 → 3.** MySQL is the order database (order state machine); Redis is the queue and session store (print jobs, session state). Failure of either breaks ordering and printing end-to-end.

### 7-Question Analysis

1. **Correct scope?** Yes — Docker compose, deployment scripts, and infra config live in `woosoo-platform`. Specialist: `infra`. ✓
2. **Already exists?** PLT-CASE-006 (COMPLETE) fixed the orchestration layout. This is a runtime failure of the running stack, not a layout issue. Not a duplicate. ✓
3. **Scope right?** Broader than the slug suggests. Two distinct root causes identified:
   - **(A) `REVERB_HOST=0.0.0.0` bug in `apply-woosoo-config.sh` line 344.** `0.0.0.0` is the Reverb server's bind address — not a valid connection target. The Laravel broadcast client needs `REVERB_HOST=reverb` (Docker internal service DNS). The prod log (`http://192.168.100.7:8080`) confirms the env was generated by an older version of the script (or never regenerated) that used the Pi LAN IP. Both `0.0.0.0` and the LAN IP are wrong for intra-container broadcast.
   - **(B) MySQL/Redis container health unknown.** The 461-error cluster and Reverb cURL 7 may be downstream of containers being unhealthy or the stack started in a bad state. Cannot verify from this repo — requires on-device `docker compose ps` and `docker compose logs mysql redis`. Config values in `apply-woosoo-config.sh` (`DB_HOST=mysql`, `REDIS_HOST=redis`) are correct Docker service names.
4. **What breaks if wrong?** MySQL down → 500 on all order/session endpoints. Redis down → queue, print jobs, sessions, Reverb pub/sub all fail. REVERB_HOST wrong → realtime broadcast silently fails (Pusher cURL 7). Combined: restaurant ordering is fully broken.
5. **Simpler path?** Fix (A) is a one-line change in `apply-woosoo-config.sh` (`REVERB_HOST` → `reverb`). Fix (B) requires on-device diagnosis; cannot be resolved from config alone.
6. **Touches contract/auth/state machine?** YES — order state machine (MySQL), print job queue (Redis), Reverb auth (Redis session). **Tier 3.** ✓
7. **Split required?** No — both issues are infra config in the same deployment stack. REVERB_HOST fix is in `apply-woosoo-config.sh`. Container health diagnosis is compose/deploy concern. One infra case handles both.

**Verdict: PROCEED — Tier 3. Specialist: infra.**

**Primary fix:** `scripts/deployment/apply-woosoo-config.sh` line 344: `REVERB_HOST=0.0.0.0` → `reverb`. Secondary: on-device container health verification required before confirming MySQL/Redis are actually healthy.

Candidate skills: `docker-deployment-debug`
