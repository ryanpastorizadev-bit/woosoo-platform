===============================================================================

PACKAGES ENDPOINT:
  GET /api/v2/tablet/packages
  - Returns only active Package rows
  - No legacy fallback IDs (46/47/48)
  - Response: Package[] (no legacy merge)

PACKAGE DETAILS:
  GET /api/v2/tablet/packages/{id}
  - Returns PackageDetails with drinks key (not beverage)
  - Excludes invalid modifiers
  - Changes response key: beverage → drinks

CATEGORY MENUS:
  GET /api/v2/tablet/categories/{slug}/menus
  - Fixed POS group ID mapping:
    meats: 34
    sides: 29
    drinks: 30
  - No fallback chain, 422 for invalid slug

ORDER SUBMISSION:
  Payload unchanged: { guest_count, package_id, items: [{menu_id, quantity}] }
  Menu validation includes meats and drinks separately
  Backend validates menu_id existence
