CONTRACT IMPACT
================================================================================

BREAKING CHANGES:
1. PackageDetails response key: beverage→drinks
   - Impact: Any client reading limits.beverage will fail
   - Mitigation: Requires simultaneous C1 + C2 deployment

2. Meats availability logic
   - Old: Meats shown only if package selected (empty otherwise)
   - New: All group-34 meats shown, disabled unless in package modifiers
   - Impact: UI displays "what's available" independent of order state

3. No legacy package ID support
   - IDs 46/47/48 no longer work without Package configuration
   - Recovery: Admin must create Package rows with krypton_menu_id

NON-BREAKING:
1. Order submission payload unchanged
   - Still: { guest_count, package_id, items: [{menu_id, quantity}] }
   - Backward compatible with existing backend validation

2. Menu loading parallelized
   - Packages, meats, sides, drinks all fetch concurrently
   - Performance improvement, no behavior change

3. Persist version migration
   - localStorage automatically cleared (version 1→2)
   - Users won't experience stale data pollution
